** what is a VOLATILE keyword and when is it used? ** 
    for Single threaded operations
        it is used to tell the compiler to not update or optimize a specific
    variable. It is used for variables that can be changed later outside of the normal program flow, meaning it can change due to hardware, interrupts or shared memory.
    Without it, the compiler might cache or optimize during repeated reads assuming nothing else can change it. 

    in a multithreaded operation 
        while it ensures fresh reads, it dont guarantee thread synchronization. Volatile alone cannot precent race conditions. In this case, you'd pair volatile with thread synchronizations like mutexes or atomics to ensure both visibilty and ordering of operations

** what is an INTERRUPT **
    Interrupts are signals that tell the PROCESSOR that something important has happened and it's immediate attention is required. These signals can be triggered by hardware like mouse move, key pressed, etc and they can be generated by the processor core itself, for example a page fault signal or a division by zero function are also considered as interrupts. These are used very often and modern CPUs are able to handle hundreds to thousands of interrupts per second. This can replace polling since polling is wasting memory on performing its main task (polling) at every nth second which can be a wasted work if it finds nothing. 

    These INTERRUPTS are pre defined function that are called whenever an interrupt signal reaches the processor. They are built between the processor and the OS. These are called INTERRUPT SERVICE ROUTINES (ISR). Every single interrupt has its own dedicated ISR, for example Page Faults has its own unique ISR and so on.
